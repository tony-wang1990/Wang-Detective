# ⚠️ 加密密钥文件

此文件保存了 King-Detective 用于加密敏感数据的密钥。

## 重要说明

1. **请勿删除此文件** - 删除将导致已加密的数据无法解密
2. **请勿提交到 Git** - 此文件已在 .gitignore 中排除
3. **备份此文件** - 建议定期备份到安全位置
4. **生产环境** - 建议使用环境变量 `KING_DETECTIVE_SECRET_KEY` 而非文件

## 优先级

系统会按以下优先级读取密钥：
1. 环境变量 `KING_DETECTIVE_SECRET_KEY`（推荐生产环境使用）
2. 当前目录下的 `.secret_key` 文件
3. 自动生成新密钥并保存到此文件

## 生产环境配置

Docker:
```bash
docker run -e KING_DETECTIVE_SECRET_KEY="your-32-character-secret-key" ...
```

Docker Compose:
```yaml
environment:
  - KING_DETECTIVE_SECRET_KEY=your-32-character-secret-key
```

## 密钥要求

- 长度：至少 32 字符（推荐）
- 字符：字母、数字、特殊字符混合
- 强度：越复杂越安全
