# Wang-Detective UI 重设计路线

更新时间：2026-05-13

本文件记录新版 UI 的既定方向、当前完成进度和后续拆分步骤。后续新增功能优先按这里的框架走，避免出现功能能用但入口、风格和主面板割裂的问题。

## 已确定的新 UI 方向

| 项目 | 约定 |
| --- | --- |
| 整体风格 | 现代运维控制台，不再沿用高饱和蓝紫渐变和装饰光效 |
| 主色 | 深石墨侧边栏、浅灰工作区、白色内容面板 |
| 强调色 | 青色用于主操作和当前状态，琥珀色用于更新/提醒，红色只用于危险动作 |
| 组件形态 | 低圆角、低阴影、高信息密度，适合长期运维扫描 |
| 页面结构 | 继续保留左侧导航、顶部状态栏、右侧内容区 |
| 新功能入口 | 必须进入主面板内容区，不能默认弹出独立页面 |
| 低配 VPS | UI 优化不引入额外常驻重任务，不增加服务端压力 |

## 尚未完成

| 优先级 | 模块 | 后续要做 | 说明 |
| --- | --- | --- | --- |
| P0 | Vue 原生路由 | 恢复或重建前端 Vue 源码，把“新版功能”“运维终端”做成真正的 Vue 路由页面 | 当前仓库只有打包产物，短期先保留 iframe 嵌入作为过渡 |
| P0 | 全量页面重绘 | 任务列表、服务日志、系统配置、AI 聊天室继续按新版主题细化 | 首页和配置列表已深化，任务列表是下一步重点 |
| P1 | 运维终端组件化 | 把 SSH、SFTP、主机库、审计从独立 HTML 拆成 Vue 组件 | 完成后可去掉 iframe 过渡层 |
| P1 | 新版功能中心组件化 | 诊断、路线图、功能入口改成 Vue 页面和复用组件 | 方便后续继续扩展 |
| P1 | 交互体验 | 菜单 active 状态、页面标题、面包屑、加载态、错误态统一 | 和路由化一起处理 |
| P2 | 响应式细节 | 移动端和窄屏布局做专项调整 | 现阶段保持不破坏原有可用性 |

## 下一步建议

1. 先部署验证当前主题是否符合你确认的 mockup 方向。
2. 如果你确认视觉方向没问题，下一阶段优先处理 Vue 原生化：建立可维护的前端源码结构，而不是继续长期修改打包产物。
3. 原生化完成后，再逐页重绘功能页面，顺序建议是：首页、配置列表、任务列表、运维终端、新版功能中心、系统配置、AI 聊天室。
